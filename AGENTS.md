# 🤖 GangNiaga WebBridge — AI Agents Integration Guide

Welcome to the **Agents Integration Guide** for GangNiaga WebBridge. This document details how AI agents (including Hermes, OpenClaw, PUSPA-V4, and custom Python/Node.js LLM agents) connect, authenticate, and control the Windows Google Chrome browser through the WebBridge.

---

## 📡 1. Agent-Daemon Interaction Model

The GangNiaga WebBridge daemon acts as a translation layer between AI agents and the Chrome Extension. It exposes two communication protocols for agents:

1.  **HTTP REST API (`http://<IP>:10087`)**: Best for simple, stateless, or batch automation commands.
2.  **WebSocket Protocol (`ws://<IP>:10087/ws`)**: Best for real-time, interactive, event-driven agent loops.

```
                  ┌─────────────────────────────────┐
                  │   AI Agent (Hermes, Python)    │
                  └────────┬──────────────┬─────────┘
                           │              │
             HTTP commands │              │ WebSocket (JSON / E2EE)
                           ▼              ▼
                  ┌─────────────────────────────────┐
                  │    Node.js WebBridge Daemon     │
                  └────────────────┬────────────────┘
                                   │
                                   │ Native Messaging / WebSockets
                                   ▼
                  ┌─────────────────────────────────┐
                  │   Chrome Extension (Active)     │
                  └─────────────────────────────────┘
```

## 🔑 2. Agent Authentication

Every state-mutating API call requires the API key generated by the daemon at runtime.
*   **Locating the Key**: Read from `daemon/.webbridge-auth.json` (keys: `apiKey`).
*   **HTTP Header**:
    ```http
    Authorization: Bearer <GANGNIAGA_API_KEY>
    ```
*   **WebSocket Query Parameter**:
    ```ws
    ws://<IP>:10087/ws?token=<GANGNIAGA_API_KEY>
    ```

## 🦅 3. Hermes Agent (Out-of-the-Box CLI)

The project includes the **Hermes Agent** (`hermes-agent-wsl.js`), a pre-built CLI automation agent written in Node.js. It operates on both Windows and WSL.

### Windows Native Commands
*   **Automated Test**: `npm run hermes` (Navigates to Shopee, performs search, and saves a screenshot).
*   **Interactive Shell**: `npm run hermes:interactive` (Open-ended manual command dispatcher).

### WSL Commands (Kali / Ubuntu)
After running `wsl-setup.sh`, use the system-wide aliases:
*   `hermes` — Executes automated workflow.
*   `hermes-i` — Runs the interactive CLI shell.

*For configuration details, see [skills/hermes-agent-wsl/SKILL.md](skills/hermes-agent-wsl/SKILL.md) and [skills/hermes-agent-windows/SKILL.md](skills/hermes-agent-windows/SKILL.md).*

## 🐍 4. Custom Python Agents (LangChain, CrewAI, AutoGen)

If you are developing custom AI agents in Python, you can expose the WebBridge command suite as tools.

### Python REST Tool Execution Example
```python
import requests
import json

class WebBridgeTool:
    def __init__(self, host="127.0.0.1", port=10087, api_key=""):
        self.base_url = f"http://{host}:{port}"
        self.headers = {
            "Content-Type": "application/json",
            "Authorization": f"Bearer {api_key}"
        }

    def execute_command(self, action: str, args: dict = None) -> dict:
        url = f"{self.base_url}/command"
        payload = {
            "action": action,
            "args": args or {}
        }
        response = requests.post(url, headers=self.headers, json=payload)
        return response.json()

# Usage
bridge = WebBridgeTool(api_key="your_api_key")
result = bridge.execute_command("navigate", {"url": "https://shopee.com.my", "newTab": True})
print(result)
```

## 🔒 5. Zero-Trust Encryption Tunnel (E2EE)

When running the daemon with `GANGNIAGA_SECRET` enabled, all WebSocket data frames must be encrypted using **AES-256-GCM**.

### Python Encryption Helper (`E2EETunnel`)
```python
import base64
import os
import json
import hashlib
from cryptography.hazmat.primitives.ciphers.aead import AESGCM

class E2EETunnel:
    def __init__(self, secret_key: str):
        # Derive 32-byte key
        self.key = hashlib.sha256(secret_key.encode()).digest()
        self.aesgcm = AESGCM(self.key)

    def encrypt(self, data_dict: dict) -> dict:
        plain_text = json.dumps(data_dict).encode('utf-8')
        iv = os.urandom(12)
        ciphertext = self.aesgcm.encrypt(iv, plain_text, None)
        combined = iv + ciphertext
        base64_data = base64.b64encode(combined).decode('utf-8')
        return {"encrypted": True, "data": base64_data}

    def decrypt(self, encrypted_dict: dict) -> dict:
        if not encrypted_dict.get("encrypted"):
            return encrypted_dict
        combined = base64.b64decode(encrypted_dict["data"])
        iv = combined[:12]
        ciphertext = combined[12:]
        decrypted = self.aesgcm.decrypt(iv, ciphertext, None)
        return json.loads(decrypted.decode('utf-8'))
```

## 🧠 6. System Prompt Instructions for LLM Agents

When instructing a generative AI agent (like GPT-4, Claude, or Gemini) to use the WebBridge, append the following guidelines to its system prompt:

```text
You are an autonomous browser control agent. You can control a real Chrome browser on the host machine using the WebBridge tool.

To interact with a page, prioritize this execution strategy:
1. Query site knowledge selectors via /sites/<domain> to find pre-mapped interactive targets.
2. If selectors are not found, request a `snapshot` of the page to get the AXTree.
3. AXTree output provides interactive element references (e.g. "@e1", "@e2"). You can pass these references directly as the "selector" argument to the `click`, `fill`, and `hover` actions.
4. Always handle inputs using the `fill` action to ensure compatibility with modern reactive frameworks (React, Vue, Angular).
5. If selectors are broken, the extension will attempt to heal them automatically using local AI models. Always check the return values for new healed selectors.
```
