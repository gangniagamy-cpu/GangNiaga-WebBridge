# 🎼 GANGNIAGA WEBBRIDGE PRO — HERMES AUTOPILOT PROTOCOL

You are equipped with the **GangNiaga WebBridge Pro**, a native browser automation bridge that connects local AI agents directly to the user's active, authenticated Google Chrome profile on Windows. 

The daemon runs at `http://127.0.0.1:10087` (HTTP) and `ws://127.0.0.1:10087/ws` (WebSocket).

---

## 🔑 1. Authentication & Session Initialization

Every request to the HTTP server or WebSocket connection (unless it is the `/status` or `/health` endpoint) requires the API key generated by the daemon.

### Locating the API Key
Read the API key directly from the local JSON config file:
*   File Location: `D:\GangNiaga-WebBridge\daemon\.webbridge-auth.json`
*   JSON Key: `apiKey`

### Auth Headers for HTTP REST
```http
Authorization: Bearer <your_api_key>
x-gangniaga-token: <your_api_key>
```

### Auth Query for WebSocket
When establishing a WebSocket connection, pass the token as a query parameter:
```ws
ws://127.0.0.1:10087/ws?token=<your_api_key>
```

### 🐧 WSL (Windows Subsystem for Linux) Configuration
If running inside a WSL terminal/agent environment, the daemon runs on the Windows host, and the Windows filesystem is mounted under `/mnt/d/`:
1.  **Locating the API Key**: Read the API key from `/mnt/d/GangNiaga-WebBridge/daemon/.webbridge-auth.json`.
2.  **Daemon Endpoint**: Do NOT connect to `127.0.0.1`. Instead, determine the Windows host gateway IP inside WSL by running:
    ```bash
    HOST_IP=$(ip route show default | awk '{print $3}')
    ```
    Connect to `http://${HOST_IP}:10087` for HTTP requests, and `ws://${HOST_IP}:10087/ws?token=<your_api_key>` for WebSockets.

## 📡 2. Core REST Endpoints

1.  **Check Browser & Daemon Status (Public)**:
    ```bash
    curl -s http://127.0.0.1:10087/status
    ```
    *Returns `{ ok: true, running: true, extension_connected: true, version: "2.0", auth_active: true }`.*

2.  **Retrieve Site Knowledge / Rules (Requires Auth)**:
    ```bash
    curl -s -H "Authorization: Bearer <key>" http://127.0.0.1:10087/sites/<domain>
    ```
    *Use this to retrieve pre-mapped CSS selectors (search boxes, login buttons) to avoid scraping raw DOM trees.*

3.  **Persist Healed Selector Rules (Requires Auth - POST `/sites/update`)**:
    When a selector fails and you heal it (via local Gemini Nano or AXTree search), persist it to the local YAML database:
    ```bash
    curl -X POST http://127.0.0.1:10087/sites/update \
      -H "Authorization: Bearer <key>" \
      -H "Content-Type: application/json" \
      -d '{
        "domain": "shopee.com.my",
        "originalSelector": "search_input",
        "healedSelector": "input#search-box-new-healed",
        "healedIndex": 3,
        "tag": "INPUT"
      }'
    ```

4.  **Execute Commands (Requires Auth - POST `/command`)**:
    ```bash
    curl -X POST http://127.0.0.1:10087/command \
      -H "Authorization: Bearer <key>" \
      -H "Content-Type: application/json" \
      -d '{"action": "ACTION_NAME", "args": { ... }}'
    ```

## 🛠️ 3. Upgraded Command Action Suite

When sending commands to `POST /command`, use these whitelisted action names and arguments:

### A. Browser Navigation & Tab Control
*   `navigate` (or `open_tab`): Navigate active tab or open URL in a new tab.
    *   `args`: `{"url": "https://google.com", "newTab": false}`
*   `close_tab`: Close a specific tab.
    *   `args`: `{"_tabId": 1234}`
*   `list_tabs` (or `get_tabs`): Get information about all open tabs in the profile.
    *   `args`: `{}`

### B. Page Interaction & DOM Control
*   `click`: Click an element via CSS selector or AXTree ref.
    *   `args`: `{"selector": "button#submit"}` or `{"selector": "@e1"}`
*   `fill`: Focus and set text in input fields (React/Vue compatible value setter).
    *   `args`: `{"selector": "input[type='text']", "value": "search query"}` or `{"selector": "@e2", "value": "text"}`
*   `mouse_click`: Human-like mouse movement curves followed by click (simulated at coordinate level to bypass anti-bots).
    *   `args`: `{"selector": "button.submit-btn"}`
*   `hover`: Move cursor to element coordinate bounds.
    *   `args`: `{"selector": "@e5"}`
*   `scroll`: Scroll page by delta offset or to specific element.
    *   `args`: `{"x": 0, "y": 500}` or `{"selector": "div.footer"}`
*   `upload`: Handle file upload inputs.
    *   `args`: `{"selector": "input[type='file']", "files": ["C:\\path\\to\\file.png"]}`

### C. Text, Keyboard & Typing Automation
*   `key_type`: Type a string character-by-character with realistic delays and self-correcting typos.
    *   `args`: `{"text": "Hello World"}`
*   `send_keys`: Send keyboard modifiers, shortcut combinations, or key sequences.
    *   `args`: `{"keys": "Control+a"}` or `{"keys": "Enter"}` (modifiers include Alt, Ctrl, Shift, Meta, Mod)
*   `extract_text`: Get raw `innerText` of the page body.
    *   `args`: `{}`

### D. Advanced Browser Tools (CDP Level)
*   `snapshot`: Get the simplified Accessibility Tree (AXTree) representing interactive page elements with references (like `@e1`, `@e2`).
*   `screenshot`: Take a DOM-level screenshot of the tab or a specific element.
    *   `args`: `{"format": "png", "selector": "div#chart"}` (selector optional)
*   `save_as_pdf`: Render the page layout as a PDF.
    *   `args`: `{"paper_format": "a4", "scale": 1.0, "landscape": false, "print_background": true}`
*   `evaluate`: Execute custom Javascript securely in the page's execution context.
    *   `args`: `{"code": "document.querySelector('h1').textContent;"}`
*   `network`: Capture real-time HTTP requests, response statuses, and payloads.
    *   `args`: `{"cmd": "start"}` (starts capturing), `{"cmd": "list"}` (lists captured), `{"cmd": "detail", "requestId": "req1"}` (gets response body), `{"cmd": "stop"}` (stops)
*   `cdp`: Raw Chrome DevTools Protocol command passthrough.
    *   `args`: `{"method": "Page.reload", "params": { "ignoreCache": true }}`

### E. OS-Level Native Automation (via PowerShell)
*   `os_screenshot`: Capture full screenshot of the primary monitor.
    *   `args`: `{"path": "D:\\temp\\screenshot.png"}`
*   `os_click`: Perform OS-level cursor click at exact coordinates.
    *   `args`: `{"x": 450, "y": 800}`
*   `hotkey`: Press system-wide OS key combinations.
    *   `args`: `{"keys": ["alt", "tab"]}`

## 🛜 4. WebSocket Bidirectional Protocol

To build a real-time event-driven agent loop, connect to `ws://127.0.0.1:10087/ws?token=<key>`.

### Client Handshake
Once the socket opens, send a `hello` handshake message:
```json
{
  "type": "hello",
  "payload": {
    "extensionVersion": "1.9.15"
  }
}
```
The daemon responds with `hello_ack` to confirm authorization.

### Executing Actions over WebSocket
Send a `tool_call` frame containing the request ID and action payload:
```json
{
  "type": "tool_call",
  "requestId": "unique-req-id-123",
  "payload": {
    "name": "navigate",
    "args": { "url": "https://google.com" }
  }
}
```
Wait for the daemon to return the matching `tool_result` frame:
```json
{
  "type": "tool_result",
  "responseToRequestId": "unique-req-id-123",
  "payload": {
    "success": true,
    "tabId": 1234
  }
}
```

## 🔒 5. End-to-End Encryption (E2EE) Mode

If the environment variable `GANGNIAGA_SECRET` is active in the daemon configuration, all WebSocket communication is encrypted using **AES-256-GCM**.

### Tunnel Protocol
1.  **Encryption Key**: Derive a 256-bit key by hashing the secret with SHA-256:
    `key = sha256(GANGNIAGA_SECRET)`
2.  **Payload format**: Encrypted payloads are sent as JSON objects with the `encrypted` flag:
    `{ "encrypted": true, "data": "<base64_encoded_payload>" }`
3.  **Data buffer format**: The base64 `data` string wraps a combined buffer:
    `[ 12-byte IV ] + [ Ciphertext ] + [ 16-byte Auth Tag ]`

Use these steps to encrypt `tool_call` payloads and decrypt incoming `tool_result` or `hello_ack` frames.

## 🧠 6. The Hermes Autopilot Loop

Always follow this targeted execution sequence:

```
[Verify Status] ➔ [Fetch Site Knowledge] ➔ [Execute Actions] ➔ [Auto-Heal & Persist Rules]
```

1.  **Status Check**: Query `/status`. Confirm `extension_connected: true`.
2.  **Sites Rule Retrieval**: Retrieve selectors for the domain from `/sites/<domain>`. If present, prioritize using them to keep token usage low.
3.  **AXTree Fallbacks**: If selectors are missing, run a `snapshot` to get the accessibility tree. Use references like `@e1` directly to interact with interactive elements (inputs, links, buttons).
4.  **AI Selector Healing**: If an element's selector changes, use Chrome's local Gemini Nano to identify the new element target, perform the action, and immediately call `/sites/update` to save the rule.
5.  **Graceful Closure**: Cleanup intermediate tabs using `close_tab`.
